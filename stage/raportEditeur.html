Plateforme d'Édition d'Articles

Comment est née le projet ?

Ce projet est né du besoin d'une société d'expertise comptable qui publie régulièrement des articles sur son site web. Initialement, ces articles étaient statiquement codés en HTML, ce qui posait problème car les rédacteurs de la société ne possédaient pas de compétences en codage. Ils ont donc demandé à leur fournisseur web de développer une interface conviviale permettant de gérer dynamiquement les articles, évitant ainsi la création manuelle de pages HTML.

Cette application web, réalisée avec Flask, offre une interface intuitive aux utilisateurs pour créer, éditer et publier des articles. Elle met l'accent sur la simplicité avec un tableau de bord convivial et intègre des fonctionnalités CRUD pour une gestion facile du contenu. La sécurité est renforcée par l'authentification utilisateur, chaque utilisateur ayant un profil personnalisable, et des notifications par e-mail sont incluses pour une expérience utilisateur complète.

Les notions à avoir

  • Flask est un framework web minimaliste pour Python.
  • MailDev est un outil de développement facilitant le test des e-mails en créant un serveur SMTP local.
  • PostgreSQL est un système de gestion de base de données relationnelle open-source.
  • pgAdmin 4 est une interface open-source pour gérer les bases de données PostgreSQL.

Cahier des Charges - Application Web de Gestion d'Articles

Objectif

L'objectif principal de ce projet est de développer une application web permettant à la société d'expertise comptable de gérer dynamiquement ses articles sur le site internet. L'application doit offrir une interface conviviale, facilitant la création, l'édition, et la publication d'articles sans nécessiter de compétences en codage.

Fonctionnalités Principales

  1. Authentification Utilisateur

    • Les utilisateurs doivent avoir la possibilité de créer un compte.
    • Les utilisateurs doivent pouvoir se connecter à l'application.
    • Les utilisateurs doivent pouvoir réinitialiser leur mot de passe de façon sécurisée.

  2. Gestion des Articles

    • Les utilisateurs doivent pouvoir créer de nouveaux articles.
    • Les articles doivent pouvoir être édités et mis à jour.
    • Les utilisateurs doivent avoir la possibilité de publier ou de supprimée des articles.
    • Les articles doivent être enregistrés dans une base de données.

  3. Tableau de Bord Intuitif

    • Mise en place d'un tableau de bord facile à utiliser.
    • Affichage clair des articles publiés, avec options d'édition et de suppression.

  4. Profil Utilisateur

    • Chaque utilisateur doit avoir un profil personnalisable.

  5. Notifications par E-mail

    • Intégration de notifications par e-mail pour les actions importantes (nouvel article, édition, etc.).

  6. Sécurité

    • Mise en place d'une authentification sécurisée.
    • Protection contre les vulnérabilités connues.

Technologies Utilisées

  • Framework : Flask (pour le développement web en Python).
  • Base de données : PostgreSQL (à confirmer selon les besoins spécifiques).

Contraintes Techniques

  • L'application doit être compatible avec les navigateurs web modernes (Chrome, Firefox, Safari).
  • L'interface utilisateur doit être responsive pour une utilisation sur des appareils mobiles.

Livrables Attendus

  • Code source complet de l'application.
  • Documentation d'utilisation et de déploiement.

Calendrier

Le projet devrait être complété dans un délai de 4 semaines à compter de la date de début, avec des points de contrôle hebdomadaires pour assurer le suivi du développement.
